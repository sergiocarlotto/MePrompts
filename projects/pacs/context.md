# Contexto do Projeto — pacs

## Fluxo de análise e abertura de cards

Cards de problemas, bugs e qualquer situação reportada pelo cliente são abertos **inicialmente no projeto ANAP**.

Somente após:
1. Confirmação do problema
2. Definição do método de reprodução
3. Entendimento de que o bug deve ser corrigido ou uma melhoria deve ser executada

...é que uma tarefa deve ser aberta no **projeto AP**.

> Em outras palavras: ANAP é a triagem; AP é o trabalho confirmado.
